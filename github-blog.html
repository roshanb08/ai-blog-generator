Master Long-Horizon Document Parsing with One-Shot OCR

Stop struggling with fragmented document extraction. Discover how "one-shot long-horizon parsing" is redefining how machines read and understand complex, multi-page layouts in a single pass.

What is it?

At its core, this repository introduces a powerful Optical Character Recognition (OCR) framework designed for Long-horizon Parsing. Traditional OCR often struggles when faced with extremely long documents or high-resolution images, typically requiring the document to be chopped into small tiles or processed page-by-page, which often loses the global context and structural relationship between elements.

This project shifts the paradigm by enabling "one-shot" parsing. Instead of treating a document as a series of isolated images, it treats the entire sequence—whether it is a massive single-page infographic or a multi-page PDF—as a continuous stream of information. By leveraging a sophisticated vision-language model architecture, it can parse complex layouts, maintain structural integrity, and extract text and formatting with high precision without the need for cumbersome pre-processing pipelines.

Key Features & Use Cases

The framework is built to handle the "messy" reality of real-world documents. Here are the concrete capabilities and how developers can apply them:

  • Multi-Page PDF Analysis: Unlike standard OCR that treats every page as a new start, this system can maintain context across pages, making it ideal for analyzing legal contracts, financial reports, or academic papers.
  • High-Resolution Image Handling: Through flexible configuration (such as the "gundam" or "base" modes), the model can adapt its crop and size settings to handle everything from small receipts to massive architectural blueprints.
  • Structural Preservation: It doesn't just dump text; it parses the layout. This makes it perfect for converting complex PDFs into structured formats like Markdown or JSON for downstream RAG (Retrieval-Augmented Generation) pipelines.
  • One-Shot Inference: By reducing the need for multiple passes or complex tiling logic, it simplifies the developer experience, allowing you to send a document and receive a comprehensive parse in one operation.
Example Use Case: Imagine an AI agent that needs to read a 50-page annual report to find a specific trend. Instead of indexing 50 separate images, this tool can parse the entire horizon, ensuring that tables spanning across two pages are captured as a single logical entity.

Why is it trending?

The current AI landscape is shifting from simple text generation to Multimodal Understanding. The community is increasingly demanding tools that can "see" and "read" documents exactly as humans do. This repository has gained significant traction because it addresses a critical bottleneck: the "context window" for visual information.

While previous models could handle single images well, they often failed when the resolution became too high or the document length too great. By pushing the boundaries of long-horizon parsing, this project provides a scalable solution for enterprises dealing with massive archives of unstructured data. The rapid growth in stars and the immediate availability of a Hugging Face demo indicate a high demand for high-fidelity, end-to-end document parsing that doesn't require manual layout analysis.

Who should use it?

This tool is designed for developers and researchers working at the intersection of Computer Vision (CV) and Natural Language Processing (NLP). Specifically, it is a goldmine for:

  • LLM Engineers: Those building RAG systems who need a high-quality "ingestion" layer to turn PDFs into clean, structured text.
  • Data Scientists: Professionals automating the extraction of data from complex forms, invoices, or scientific journals.
  • Python Developers: Anyone comfortable with the transformers ecosystem and NVIDIA GPUs who wants to implement state-of-the-art document AI without building a model from scratch.

Given its reliance on torch and transformers, it fits seamlessly into the modern PyTorch ecosystem, making it accessible to anyone familiar with Hugging Face model deployment.

Getting started

To get the system running on an NVIDIA GPU, ensure you have Python 3.12.3 and CUDA 12.9 installed. Follow these steps:

1. Install the required dependencies:

pip install torch==2.10.0 torchvision==0.25.0 transformers==4.57.1 Pillow==12.1.1 matplotlib==3.10.8 einops==0.8.2 addict==2.4.0 easydict==1.13 pymupdf==1.27.2.2 psutil==7.2.2

2. Run a basic inference:

import torch
from transformers import AutoModel, AutoTokenizer

model_name = 'baidu/Unlimited-OCR'
tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)
model = AutoModel.from_pretrained(model_name, trust_remote_code=True, use_safetensors=True, torch_dtype=torch.bfloat16).eval().cuda()

model.infer(
    tokenizer,
    prompt='<image>document parsing.',
    image_file='your_image.jpg',
    output_path='your/output/dir',
    base_size=1024, image_size=640, crop_mode=True,
    max_length=32768,
    save_results=True,
)
